## 1. 概述

### 1.1 功能简介
用户每天可签到一次，获取每日签到奖励；一周内连续签到可获得额外奖励；签到记录以周为周期，每周一自动刷新重置。所有奖励金额支持后台配置。

### 1.2 核心特性
- ✅ 每日签到：每日00:00刷新，当天错过无法补签
- ✅ 周签到周期：自然周（周一至周日）
- ✅ 基础签到奖励：每日签到获得固定奖励
- ✅ 连续签到额外奖励：一周内连续签到N天，解锁额外奖励
- ✅ 后台可配置：所有奖励金额、连续签到天数阈值均支持后台配置
- ✅ 签到状态展示：日历形式展示本周7天签到状态

## 2. 业务规则详解

### 2.1 签到周期规则

#### 2.1.1 周期定义
- **周期单位**：自然周（周一至周日）
- **周期刷新时间**：每周一 00:00:00（服务器时间）
- **周期内天数**：7天

#### 2.1.2 周期刷新逻辑
```
示例场景：
- 当前时间：2026-06-09 00:00:01（周二）
- 上一周期：2026-06-02（周一）~ 2026-06-08（周日）
- 新一周期：2026-06-09（周一）~ 2026-06-15（周日）

刷新操作：
1. 标记上一周期结束
2. 重置所有用户的周签到状态
3. 初始化新周期的签到配置
```

---

### 2.2 签到规则

#### 2.2.1 每日签到规则
| 规则项 | 说明 |
|--------|------|
| 签到时间 | 每日 00:00:00 ~ 23:59:59 |
| 签到次数 | 每日1次 |
| 补签机制 | ❌ 不支持补签，错过即失去 |
| 重复签到 | 返回提示"今日已签到" |

#### 2.2.2 签到状态定义
```dart
enum CheckinDayStatus {
  locked,      // 未到日期，不可签到（未来）
  available,   // 可签到（今天且未签到）
  done,        // 已签到
  missed,      // 已错过（过去某天未签到）
}
```

---

### 2.3 奖励规则

#### 2.3.1 基础签到奖励
- **奖励名称**：每日签到奖励
- **发放时机**：用户完成签到时立即发放
- **奖励内容**：金币（coins）
- **配置方式**：按星期几配置不同金额，或统一金额

配置示例（支持灵活配置）：
```json
{
  "dailyRewards": {
    "monday": 10,
    "tuesday": 10,
    "wednesday": 20,
    "thursday": 10,
    "friday": 10,
    "saturday": 15,
    "sunday": 15
  }
}
```
或简化配置：
```json
{
  "dailyRewardDefault": 10
}
```

#### 2.3.2 连续签到额外奖励

**连续签到定义**：在一个周期（周一至周日）内，连续N天完成签到。

**奖励触发时机**：当用户完成签到，且连续签到天数达到某个阈值时，立即发放。

**奖励配置示例**：
```json
{
  "continuousBonus": [
    { "days": 3, "bonus": 30, "name": "连续3天奖励" },
    { "days": 5, "bonus": 50, "name": "连续5天奖励" },
    { "days": 7, "bonus": 100, "name": "完美周奖励" }
  ]
}
```

**奖励规则**：
- 每个阈值奖励在一个周期内仅可领取一次
- 若连续签到中断后重新开始，再次达到阈值可再次领取
- 奖励为累加式，不互斥（连续7天可同时获得3天、5天、7天的奖励）

**连续签到中断判断**：
- 错过某一天签到 → 连续签到天数重置为0
- 周一刷新周期 → 连续签到天数重置为0

---

### 2.4 完整奖励计算示例

**配置**：
- 每日奖励：10金币/天
- 连续3天奖励：30金币
- 连续5天奖励：50金币
- 连续7天奖励：100金币

**用户签到场景**：
```
周一：签到 → 获得10金币
周二：签到 → 获得10金币
周三：签到 → 获得10金币 + 连续3天奖励30金币 = 累计50金币
周四：签到 → 获得10金币
周五：签到 → 获得10金币 + 连续5天奖励50金币 = 累计120金币
周六：签到 → 获得10金币
周日：签到 → 获得10金币 + 连续7天奖励100金币 = 累计250金币
```

#### 3.1.3 用户周签到状态表（user_checkin_week_status）
> 存储用户每周的签到状态，避免实时计算

```sql
CREATE TABLE user_checkin_week_status (
  id BIGINT PRIMARY KEY AUTO_INCREMENT,
  user_id BIGINT NOT NULL COMMENT '用户ID',
  week_of_year INT NOT NULL COMMENT '年份第几周（格式：202623）',

  -- 签到状态
  checkin_days BIT(7) DEFAULT b'0000000' COMMENT '7天签到状态（位掩码，第0位=周一）',
  checkin_count INT DEFAULT 0 COMMENT '已签到天数',
  continuous_days INT DEFAULT 0 COMMENT '当前连续签到天数',

  -- 奖励领取状态
  claimed_bonus_mask INT DEFAULT 0 COMMENT '已领取的连续奖励掩码',

  -- 周期信息
  week_start DATE NOT NULL COMMENT '周开始日期（周一）',
  week_end DATE NOT NULL COMMENT '周结束日期（周日）',

  create_time DATETIME DEFAULT CURRENT_TIMESTAMP,
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

  UNIQUE KEY uk_user_week (user_id, week_of_year),
  INDEX idx_week (week_of_year)
) COMMENT '用户周签到状态表';
```

**checkin_days 位掩码说明**：
```
位索引：  0  1  2  3  4  5  6
星期：    一 二 三 四 五 六 日
```

**claimed_bonus_mask 说明**：
- 使用位掩码标记哪些连续奖励已领取
- 例如：`0b101` 表示已领取第1个和第3个连续奖励

### 3.3 核心业务逻辑

#### 3.3.1 签到执行流程

```
1. 校验用户是否已签到今日
   ├─ 已签到 → 返回错误
   └─ 未签到 → 继续

2. 获取当前生效的签到配置
   └─ 读取 checkin_config 表，找到当前生效的配置

3. 计算每日奖励
   ├─ 获取今天是星期几
   ├─ 查找配置中对应星期几的奖励
   └─ 未配置则使用默认值

4. 获取或创建用户周签到状态
   ├─ 计算当前 week_of_year
   ├─ 查询 user_checkin_week_status
   └─ 不存在则初始化新记录

5. 计算连续签到天数
   ├─ 检查昨天是否签到
   ├─ 签到了 → continuous_days + 1
   └─ 未签到 → continuous_days = 1

6. 检查连续签到额外奖励
   ├─ 遍历 continuous_bonus 配置
   ├─ 检查当前 continuous_days 是否达到阈值
   ├─ 检查该阈值奖励是否已领取（claimed_bonus_mask）
   └─ 收集符合条件的奖励

7. 更新用户周签到状态
   ├─ 设置对应天的签到位
   ├─ 更新 checkin_count
   ├─ 更新 continuous_days
   └─ 更新 claimed_bonus_mask

8. 发放奖励
   ├─ 计算总奖励金额
   ├─ 更新用户金币余额
   └─ 记录金币流水

9. 创建签到记录
   └─ 插入 user_checkin_record

10. 返回签到结果
```

---

#### 3.3.2 周签到状态初始化流程

```
输入：userId, weekOfYear

1. 计算该周的起止日期（周一至周日）
2. 创建 user_checkin_week_status 记录
   ├─ checkin_days = 0
   ├─ checkin_count = 0
   ├─ continuous_days = 0
   ├─ claimed_bonus_mask = 0
   └─ week_start, week_end = 计算值
3. 返回初始化后的状态对象
```

---

#### 3.3.3 连续签到天数计算逻辑

```
输入：userCheckinWeekStatus, today

1. 获取昨天的日期
2. 判断昨天是否在同一周内
   ├─ 同一周 → 检查周状态表中昨天的位是否为1
   └─ 不同周 → 检查上周是否有周日的签到记录

3. 如果昨天签到了
   └─ continuous_days = previous_continuous_days + 1
4. 如果昨天没签到
   └─ continuous_days = 1

5. 返回计算后的 continuous_days
```

---

#### 3.3.4 额外奖励检查逻辑

```
输入：continuousBonusConfig, claimedBonusMask, continuousDays

输出：符合条件的奖励列表

1. 初始化结果列表 empty
2. 遍历 continuousBonusConfig 中的每个配置项
   ├─ 检查 continuousDays >= item.days
   ├─ 检查 claimedBonusMask 的对应位是否为0（未领取）
   ├─ 两者都满足 → 添加到结果列表
3. 返回结果列表
```

// lib/models/reward/reward_models.dart

/// 签到配置
class CheckinConfig {
  final String configName;
  final String configCode;
  final Map<int, int>? dailyRewards; // key: dayOfWeek (1-7)
  final int dailyRewardDefault;
  final List<ContinuousBonusConfig> continuousBonus;

  CheckinConfig({
    required this.configName,
    required this.configCode,
    this.dailyRewards,
    required this.dailyRewardDefault,
    required this.continuousBonus,

## 8. 术语表

| 术语 | 说明 |
|------|------|
| Week Of Year | 年份周数，格式：yyyyWW（如202623表示2026年第23周） |
| Day Of Week | 星期几，1=周一，7=周日（ISO标准） |
| Continuous Days | 连续签到天数（一个周期内） |
| Bonus Mask | 奖励掩码，使用位运算标记已领取的奖励 |
| Checkin Days Bitmask | 签到位掩码，7位分别表示周一到周日的签到状态 |
